Before/After Slider — What's Wrong
3 specific bugs breaking the interaction
Fix These ▶
1
Images are missing
before1.png and after1.png don't exist yet — the slider renders as two empty gray boxes. Nothing to drag. Upload real before/after project photos and name them correctly.
2
The project card has a fixed height of 400px
The .project-card CSS sets height:400px. The ba-slider inside is position:relative with height:100% — but its parent is a grid cell, not the card. The slider collapses. Fix: add height:400px directly on .ba-slider-wrap and .ba-slider.
3
mousemove fires on window — drags outside container
The window-level mousemove listener means the slider keeps moving even when the user drags outside the card. Limit the listener to the slider element itself. Also: the overlay text sits on top with pointer-events:none but still visually confuses mobile users.
CSS fix — add to style.css
/* Fix slider height */ .ba-slider-wrap, .ba-slider { height: 400px; width: 100%; } /* Remove "Interactive Before & After" overlay text — confusing */ .ba-slider-wrap .project-overlay { display: none; }
JS fix — replace the slider block in main.js
const slider = document.getElementById('ba1'); if (slider) { let isDown = false; const afterImg = slider.querySelector('.img-after'); const handle = slider.querySelector('.handle'); const move = (e) => { if (!isDown) return; const rect = slider.getBoundingClientRect(); const x = (e.touches ? e.touches[0].clientX : e.clientX) - rect.left; const pct = Math.max(0, Math.min(x / rect.width * 100, 100)); afterImg.style.width = pct + '%'; handle.style.left = pct + '%'; }; slider.addEventListener('mousedown', e => { isDown = true; move(e); }); slider.addEventListener('touchstart', e => { isDown = true; move(e); }); slider.addEventListener('mousemove', move); // scoped to slider only slider.addEventListener('touchmove', move); window.addEventListener('mouseup', () => isDown = false); window.addEventListener('touchend', () => isDown = false); }
